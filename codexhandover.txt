Security/Policy Notes (enterprise)
- Some domains enforce Defender ASR rule 01443614-CD74-433A-B99E-2ECDC07BFC25 (prevalence/age/trusted). If the EXE is blocked, either run the DLL via dotnet host, or add an ASR OnlyExclusion for the publish folder/EXE, or code-sign and allow by publisher.

Release 1.1.7 Highlights
- Accent color system with bold Light/Dark tints; “Default” returns to classic look (blue primary only).
- Multi-file sources:
  - Add via Browse (multi-select) or drag/drop onto the Source field.
  - TextBox shows inline, comma-separated basenames with “+N more” when long; full list on tooltip.
  - New buttons on the same row: Remove (menu to remove specific file), Clear (clear all).
  - Right-click Source field shows the same menu (Add/Remove/Clear).
- Auto-clear toggles for Sources and Destinations after non-preview copy.
- Dark mode checkbox/radio backgrounds use parent BG (no grey highlight).
- Range Helper group height reduced (less blank space).
- Bottom action buttons (Undo/Redo/Engage) aligned flush-right and resilient to resize.

Code Map / Key Touchpoints (Program.cs)
- Source row: txtSource + btnSrcRemove + btnSrcClearAll + Browse. ContextMenuStrip on txtSource for managing files.
- Inline source display: RefreshSourceText() + ComposeInlineList() build a single-line summary + tooltip.
- Copy logic: GetSelectedSources() iterates each source per destination; numbered prefix/suffix increment per destination (intentional).
- LayoutBottom(): aligns action buttons to the right; keeps log/status positions dynamic.
- Theme.Resolve(): tints Light/Dark palettes by accent; “Default” leaves classic palettes.
- Checkbox/Radio theming: background matches parent (fixes “grey box” in dark mode).

Known / Follow-ups
- Build warnings:
  - CS8604 (Path.Combine param) and CS8620 (nullability on ComposeInlineList input) are benign; consider tightening nullability annotations later.
- Inline source interaction could be enhanced to allow click-to-remove on a specific token in the TextBox; current approach uses a Remove menu to keep the single-line look.
- Consider code-signing for future releases and enabling a Publisher allow rule in enterprise environments.
- If publishing while the app is running, close ManyCopy first to avoid file-lock errors.
